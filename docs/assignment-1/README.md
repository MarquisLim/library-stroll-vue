# Задание 1. Проектирование многоуровневой модели предметной области с расширенной семантикой

## Описание задания

**Цель:** Разработать полную систему моделей (от концептуальной до реализации) с применением принципов многоуровневого моделирования (M3–M0).

**Предметная область:** Library Stroll - Платформа для художников

## Структура проекта

```
assignment-1/
├── README.md                    # Этот файл
├── REPORT.md                    # Полный отчет (см. report/report-example.md)
├── m3-linguistic/              # M3: Лингвистическая метамодель
│   └── description.md         # Описание формальной нотации
├── m2-ontological/             # M2: Онтологическая метамодель
│   └── description.md         # Правила построения моделей
├── m1-conceptual/              # M1: Концептуальная модель
│   ├── description.md         # Описание модели Library Stroll
│   └── library_stroll.json    # JSON модель (ссылка на case-tool/)
├── m0-data-model/              # M0: Модель данных
│   └── description.md         # Реализация в Laravel
├── case-tool/                  # CASE-инструмент
│   ├── description.md         # Описание инструмента
│   └── structure.md           # Архитектура инструмента
├── diagrams/                    # Диаграммы
│   ├── hierarchy.md           # Четырехуровневая иерархия
│   └── er-model.md            # ER-диаграмма M1
└── report/                     # Материалы для отчета
    ├── report-example.md       # Пример отчета
    └── INSTRUCTIONS.md         # Инструкции по выполнению
```

## Быстрый старт

1. **Изучите структуру:** Начните с `report/INSTRUCTIONS.md`
2. **Просмотрите уровни:** M3 → M2 → M1 → M0
3. **Запустите CASE-инструмент:** см. `case-tool/description.md`
4. **Создайте отчет:** Используйте `report/report-example.md` как шаблон

## Ссылки на материалы

- **CASE-инструмент:** `case-tool/main.py` (в корне проекта)
- **JSON модель:** `case-tool/models/library_stroll.json` (в корне проекта)
- **Laravel миграции:** `database/migrations/` (в корне проекта)
