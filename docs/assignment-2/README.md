## Предметная область

**Проект:** Library Stroll - Платформа для художников

**Основные компоненты системы:**
- Frontend (Vue.js + Inertia.js)
- Backend API (Laravel 11)
- База данных (MySQL/SQLite)
- Файловое хранилище (Media Library)
- Real-time коммуникация (Pusher/Reverb)
- Очереди задач (Queue)

## Цель задания

Продемонстрировать способность адаптировать архитектуру и процесс разработки под разные модели жизненного цикла:
1. Каскадная (Waterfall)
2. Спиральная (Spiral)
3. Agile/Iterative

## Требования к результатам

Для каждой модели:
- ✅ Описание этапов, процессов, ролей и артефактов (ISO/IEC 12207)
- ✅ Диаграмма активностей UML (взаимодействие процессов)
- ✅ Диаграмма компонентов (архитектурные компоненты)
- ✅ Диаграмма развертывания (физическая архитектура)
- ✅ Метрики риска, стоимости и гибкости
- ✅ Имитационное моделирование изменений требований

Итоговые результаты:
- Таблица сравнения моделей
- UML-диаграммы с легендами
- Обоснованный выбор оптимальной модели
