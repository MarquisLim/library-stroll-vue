# Таблица сопоставления элементов DFD и UML

## Описание

Данная таблица показывает соответствие между элементами DFD (Data Flow Diagram) и элементами UML (Unified Modeling Language).

## Таблица сопоставления

| Элемент DFD | Элемент UML | Тип соответствия | Пример |
|-------------|-------------|------------------|--------|
| **Процесс (Process)** | Класс (Class) | 1:1 | Процесс "2.1 Валидация файла" → Класс `FileValidator` |
| **Процесс (Process)** | Метод класса (Method) | 1:1 | Процесс "2.2 Обработка изображения" → Метод `processImage()` |
| **Поток данных (Data Flow)** | Параметр метода (Parameter) | 1:1 | Поток "Файл" → Параметр `file: File` |
| **Поток данных (Data Flow)** | Возвращаемое значение (Return) | 1:1 | Поток "Обработанный файл" → Возврат `ProcessedFile` |
| **Хранилище данных (Data Store)** | Класс (Class) | 1:1 | Хранилище "База данных" → Класс `Database` |
| **Хранилище данных (Data Store)** | Атрибут класса (Attribute) | 1:1 | Хранилище "Файловое хранилище" → Атрибут `storage: Storage` |
| **Внешняя сущность (External Entity)** | Класс (Class) | 1:1 | Сущность "Пользователи" → Класс `User` |
| **Внешняя сущность (External Entity)** | Актер (Actor) | 1:1 | Сущность "Администраторы" → Актер `Administrator` |
| **Связь процесса с хранилищем** | Ассоциация (Association) | 1:1 | Процесс → Хранилище → Класс → Класс (ассоциация) |
| **Последовательность процессов** | Диаграмма последовательности (Sequence) | 1:1 | Цепочка процессов → Последовательность вызовов методов |
| **Условие (Condition)** | Условие в диаграмме активности (Guard) | 1:1 | Условие "Файл валиден?" → Guard `[file.isValid()]` |

## Правила трансформации

### Правило 1: Процесс → Класс
Если процесс имеет состояние или данные, он становится классом.  
Если процесс - простая операция, он становится методом.

### Правило 2: Поток данных → Параметр/Возврат
Входной поток данных → параметр метода  
Выходной поток данных → возвращаемое значение

### Правило 3: Хранилище → Класс/Атрибут
Хранилище данных → класс с методами доступа  
Хранилище как часть процесса → атрибут класса

### Правило 4: Последовательность → Диаграмма последовательности
Цепочка процессов → последовательность вызовов методов между объектами

### Правило 5: Условие → Guard/Decision
Условие в DFD → guard в диаграмме активности или decision node
