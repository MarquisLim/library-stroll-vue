# Эксперимент: Изменение метамодели без перекомпиляции

## Цель эксперимента

Продемонстрировать, как система адаптируется к изменению метамодели без перекомпиляции приложения.

## Сценарий эксперимента

### Шаг 1: Исходное состояние

Система имеет метаданные для сущности Artwork с базовыми атрибутами:
- id
- title
- description
- file_path
- created_at

### Шаг 2: Изменение метамодели

Добавляем новый атрибут `tags` (теги) в метаданные через SQL запрос.

### Шаг 3: Проверка адаптации

Система автоматически:
- Обновляет форму создания/редактирования
- Обновляет таблицу списка работ
- Добавляет валидацию для нового поля

## Файлы эксперимента

- `before.md` - Состояние до изменения
- `change.sql` - SQL запрос для изменения метамодели
- `after.md` - Состояние после изменения
- `demo.md` - Демонстрация работы
