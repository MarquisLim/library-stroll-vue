# Задание 4. Создание системы с управлением метаданными (DSM-подход)

## Описание задания

**Цель:** Реализовать динамически управляемую информационную систему, в которой модель определяет поведение приложения.

**Предметная область:** Library Stroll - Платформа для художников

## Структура проекта

```
assignment-4/
├── README.md                    # Главный файл с описанием
├── REPORT.md                    # Отчет по заданию
├── prototype/                   # Прототип системы
│   ├── database/               # Миграции и схемы БД
│   │   ├── migrations/        # SQL миграции
│   │   └── schema.sql         # Схема БД метаданных
│   ├── backend/                # Backend (Laravel/PHP)
│   │   ├── app/               # Код приложения
│   │   └── routes/            # Маршруты API
│   └── frontend/              # Frontend (Vue.js)
│       └── components/       # Динамические компоненты
├── metamodel/                  # Онтологическая метамодель
│   ├── ontology.md           # Описание метамодели
│   └── schema.json           # JSON схема метамодели
├── interpreter/               # Интерпретатор метамодели
│   ├── interpreter.php       # PHP интерпретатор
│   └── README.md             # Инструкции
├── uml-diagrams/              # UML диаграммы
│   ├── component-diagram.md  # Диаграмма компонентов
│   ├── class-diagram.md      # Диаграмма классов метамодели
│   └── activity-diagram.md   # Диаграмма активности интерпретации
└── experiment/                # Эксперимент
    ├── before.md             # Состояние до изменения
    ├── change.md             # Изменение метамодели
    └── after.md              # Состояние после изменения
```

## Требования

1. ✅ Реализовать прототип метаданных-управляемой системы
2. ✅ Модель хранится в БД (сущности, атрибуты, связи)
3. ✅ Автоматическое формирование интерфейсов на основе метаданных
4. ✅ Разработать онтологическую метамодель
5. ✅ Внедрить интерпретатор метамодели
6. ✅ Создать UML-диаграммы
7. ✅ Провести эксперимент с изменением метамодели
