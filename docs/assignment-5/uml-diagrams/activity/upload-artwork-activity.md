# Activity диаграмма - Загрузка работы

## Описание

Диаграмма активности показывает бизнес-процесс загрузки художественной работы художником.

## Диаграмма (Mermaid)

```mermaid
flowchart TD
    Start([Художник начинает загрузку]) --> SelectFile[Выбрать файл]
    SelectFile --> ValidateFile{Файл<br/>валиден?}
    
    ValidateFile -->|Нет| ErrorFile[Ошибка:<br/>Неверный формат]
    ErrorFile --> SelectFile
    
    ValidateFile -->|Да| EnterMetadata[Ввести метаданные:<br/>Название, описание]
    EnterMetadata --> AddTags[Добавить теги]
    AddTags --> SetPrivacy[Настроить приватность:<br/>Публичная/Приватная]
    SetPrivacy --> SetPermissions[Настроить права:<br/>Разрешить комментарии,<br/>Разрешить скачивание]
    
    SetPermissions --> ValidateMetadata{Метаданные<br/>валидны?}
    ValidateMetadata -->|Нет| ErrorMetadata[Ошибка:<br/>Заполните обязательные поля]
    ErrorMetadata --> EnterMetadata
    
    ValidateMetadata -->|Да| UploadFile[Загрузить файл<br/>на сервер]
    UploadFile --> ProcessFile[Обработать файл:<br/>Создать превью,<br/>Оптимизировать]
    
    ProcessFile --> SaveDB[Сохранить запись<br/>в БД]
    SaveDB --> LinkTags[Связать теги<br/>с работой]
    LinkTags --> NotifyFollowers[Уведомить подписчиков<br/>о новой работе]
    
    NotifyFollowers --> Success([Работа успешно<br/>загружена])
    
    style Start fill:#e1f5ff
    style Success fill:#c8e6c9
    style ErrorFile fill:#ffebee
    style ErrorMetadata fill:#ffebee
    style UploadFile fill:#fff3e0
    style ProcessFile fill:#f3e5f5
```

## Описание процесса

### Этап 1: Подготовка файла
1. Выбор файла (изображение, видео, GIF)
2. Валидация формата и размера файла

### Этап 2: Ввод метаданных
1. Ввод названия работы
2. Ввод описания (опционально)
3. Добавление тегов для категоризации
4. Настройка приватности (публичная/приватная)
5. Настройка прав (комментарии, скачивание)

### Этап 3: Загрузка и обработка
1. Загрузка файла на сервер
2. Обработка файла (создание превью, оптимизация)
3. Сохранение метаданных в БД
4. Связывание тегов с работой

### Этап 4: Уведомления
1. Уведомление подписчиков о новой работе

## Альтернативные потоки

- **Неверный формат файла** → возврат к выбору файла
- **Невалидные метаданные** → возврат к вводу метаданных
- **Ошибка загрузки** → отображение ошибки пользователю
